# data/encodings/

GeoParquet 2.0 has a single encoding (`"WKB"`); the variation here is whether the Parquet column uses the native Geometry or native Geography logical type. The native-geometry (planar `OGC:CRS84`) files are generated by this script (`gen_encodings.py`, via pyarrow). The native-geography files (Parquet native Geography logical type, spherical edges) are generated by `gen_geography.py` using Apache sedonadb — the only tool in our stack that emits that logical type — and are committed as snapshots (see `scripts/README.md`, "Geography tier").

|---|---|---|---|---|
| `point-native-geometry.parquet` | Point | native-geometry | OGC:CRS84 | planar |
| `point-native-geography.parquet` | Point | native-geography | OGC:CRS84 | spherical |
